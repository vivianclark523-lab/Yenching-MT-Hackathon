## 状态机剧本（Demo 时间轴）

### 基准：18:00 起点

所有排队初始值从 **虚拟时间 18:00** 开始计算。

### shop-001 海底捞（单调推进型 + 事件触发型）

```json
{
  "state_machine": {
    "type": "monotonic_decay",
    "params": { "initial_queue": 30, "rate_per_minute": 0.5, "t0": "18:00" }
  },
  "events": [
    { "time": "18:25", "event": "jump", "delta": -5, "reason": "前台放开双人桌" },
    { "time": "19:10", "event": "surge", "delta": +8, "reason": "下班人流涌入" }
  ]
}
```

**关键时刻**：
- 18:00 → 30 桌，管家提示"等位较长，先盯着"
- 18:25 → 突然跳到 ~18 桌（事件），管家触发阈值提醒（若阈值设为 20）
- 18:50 → 自然衰减到约 5 桌，取号+出发提醒

### shop-002 大鸭梨（时段周期型）

```json
{
  "state_machine": {
    "type": "periodic",
    "params": {
      "segments": [
        { "hour_range": [18, 19], "queue": 15 },
        { "hour_range": [19, 20], "queue": 25 },
        { "hour_range": [20, 21], "queue": 10 },
        { "hour_range": [21, 23], "queue": 3 }
      ]
    }
  }
}
```

**关键时刻**：
- 18:00 只有 15 桌，是多店对比时的"备用推荐"
- 19:00 涌到 25 桌，若用户晚出发有压力

### shop-003 外婆家（快速衰减，适合"今晚随便吃"场景）

```json
{
  "state_machine": {
    "type": "monotonic_decay",
    "params": { "initial_queue": 12, "rate_per_minute": 0.8, "t0": "18:00" }
  }
}
```

18:15 就可以取号（约 3 桌），适合演示"快速命中阈值"的 demo 路径。

## 推荐演示剧本（评委可直接用的 Prompt）

**Prompt A（双店并行，Demo 主路径）**：
> 今晚想吃火锅，海底捞和凑凑都行，帮我盯一下哪家先排到

**Prompt B（快速命中，适合 3 分钟 demo）**：
> 帮我看看外婆家排队多久，我在望京

**Prompt C（主动出击路径 D）**：
> （管家主动推）：「您上周提过想去大鸭梨，今晚 7 点前排队比较少，要帮您盯一下吗？」

**Prompt D（多路径联动）**：
> （Skill 3 规划出海底捞后）：「已同步帮您启动海底捞排队监控 🔔」
