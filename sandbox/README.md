# 虾蜜沙盒控制台 · 如何体验

一个本地、零依赖、零网络的可视化控制台：左边拨"世界态"（虚拟时钟 / 想吃什么 / 会员 / 天气 / 盯哪几家店 / 今晚行程），右边实时看三个 Skill 的反应（排队速度、跨店比价、采购比价、今晚行程）。**评委拨时间、换 case，面板立刻重算。**

> 它只读各 Skill 现成的 CLI 取数据，不重写业务逻辑；所有数据来自本地 `mocks/`，**不连任何外网 / API**。

---

## 一键启动

```bash
./sandbox/run.sh
```

这条命令自动做好四件最容易翻车的事，然后帮你开好浏览器：

1. **杀掉旧 server 进程** —— "改了没生效 / 下拉只有一条" 的头号根因（旧进程占着端口跑老代码）
2. **复位虚拟时钟到 demo 基线**（2026-06-07 18:00）—— 每次从同一时刻开始
3. 后台起 server 并等就绪
4. **自检"跑的是新代码"** + 自动打开 http://127.0.0.1:8765

可选：

```bash
./sandbox/run.sh 8770       # 换端口
NO_OPEN=1 ./sandbox/run.sh  # 不自动开浏览器
pkill -f sandbox/server.py  # 停止
```

> 要求：Python 3、在仓库根目录下、当前分支含最新沙盒代码。无需装任何依赖。

## 启动后看到什么

**左侧 = 控制区（评委拨这里）**
- ⏰ 虚拟时钟：场景预设下拉 + 时间轴滑块 + 关键时刻书签 + 复位
- 🍽️ 想吃什么（Skill 2 跨店比价品类）
- 🎭 用户 / 意图：会员开关、目标（达标最省 / 全场最低）、达标线、预算、天气
- 🍲 盯哪几家店（Skill 1 排队，可临场加店减店）
- 🗺️ 今晚行程·选预设（Skill 3，三条预设切换）
- ⚙️ 高级调参：逐店/逐券覆盖 + 事件注入

**右侧 = 效果区（实时重算）**
- 🍜 Skill 1 排队速度 / 取号 ETA
- 🎟️ Skill 2 跨店比价 + 主动推送
- 🛒 Skill 2 采购比价（券后单价 + 凑一单）
- 🗺️ Skill 3 今晚行程（沿途排队 × 券 × 余票）

顶部还有一条"因果带"：显示"评委刚做了什么 → 哪个面板因此变化"。

## 演示动线（投屏操作建议）

1. 选一个**场景预设**（如"午市神券"或"晚高峰"），一键把整个世界态拨到位。
2. **拨时钟**演示张力：余票递减（手慢无）、排队桌数跳变、神券限时激活。
3. **换品类 / 切行程预设**演示"服务找人"：同一时刻不同需求，管家给的方案不同。
4. 需要时开 ⚙️ 高级调参，**当着评委的面**改某家店的排队速度 / 注入突发事件，看面板即时反应。

## 出问题怎么办

绝大多数"看起来像 bug"其实是**旧 server 进程**。`run.sh` 已自动处理；若手动起过、怀疑没生效：

```bash
pkill -f sandbox/server.py          # 杀干净
lsof -nP -i :8765                   # 确认端口空了
./sandbox/run.sh                    # 重来
# 浏览器 Cmd+Shift+R 硬刷新
```

一行自检（确认跑的是新代码）：

```bash
curl -s http://127.0.0.1:8765/api/clock | grep -q itineraries && echo NEW || echo "STALE-重启"
```
